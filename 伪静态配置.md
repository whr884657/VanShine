# VanShine 伪静态 / URL 重写配置

VanShine **不需要**为分享功能单独配置 Nginx 规则。分享走 PHP 文件 + PATH_INFO，与站点其余 PHP 页面使用同一套解释器（PHP 7.4 / 8.0 / 8.2 均可，由面板自动处理）。

---

## 分享链接格式

| 用途 | 地址 |
|------|------|
| 安全提醒（无 token） | `{域名}/d/index.php` |
| 公开分享页 | `{域名}/d/index.php/{32位token}` |
| 下载/预览流 | `{域名}/d/index.php/{token}/stream?file=ID` |

token 在 URL 路径中，**无需**在伪静态里写 token 规则，也**无需** `?token=` 查询参数。

---

## Nginx

只需保留站点**通用**伪静态（与安装向导、后台等页面相同）：

```nginx
location / {
    try_files $uri $uri/ $uri.php$is_args$args;
}
```

PHP 解释由面板/环境自带的 `location ~ \.php` 配置负责，**不要**在配置里写死 `enable-php-82.conf` 等版本号。

### 切勿添加以下错误配置

```nginx
# 错误示例：会导致 /d/index.php 被当作静态文件直接下载源码
location ^~ /d/ {
    try_files $uri $uri/ /d/index.php$is_args$args;
}
```

若曾添加过 `/d/` 专用 `location`，请删除后只保留上面的通用 `location /`，并重载 Nginx。

---

## Apache

根目录 `.htaccess` 仅禁止访问 `config/`。`d/.htaccess` 已启用 `AcceptPathInfo`，支持 `/d/index.php/{token}` 路径格式。需启用 `mod_rewrite` 且 `AllowOverride All`。

---

## 验证

1. `https://你的域名/d/index.php` → HTML 安全提醒页（不是下载 `.php` 文件）
2. `https://你的域名/d/index.php/{token}` → 分享页
3. 后台创建分享后复制的链接可直接打开

---

## 常见问题

**Q：访问 `/d/index.php` 变成下载文件？**  
通常是 Nginx 为 `/d/` 单独加了 `try_files $uri`，命中物理文件后未交给 PHP。删除该规则，改用通用伪静态 + 面板默认 PHP 配置即可。

**Q：旧版 `?token=` 或 `/d/{token}` 链接还能用吗？**  
`?token=` 会自动 301 跳转到 `/d/index.php/{token}`。纯 `/d/{token}`（无 index.php）需 Web 服务器 rewrite，VanShine 不再依赖该格式。
