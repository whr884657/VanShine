# VanShine 伪静态 / URL 重写配置

VanShine 依赖 Web 服务器的 URL 重写，以实现：

- 隐藏 `index.php`（如 `/admin/files/manage.php` → 可选简化）
- **分享短链接** `/d/{token}` 与 `/d/{token}/stream?file=ID`

若未正确配置，访问 `https://域名/d/xxxxxxxx` 会出现 **404**。

---

## Nginx（推荐）

在 `server { ... }` 内，**将以下规则放在通用 `location /` 之前**：

```nginx
# VanShine 分享短链接
location ~ ^/d/([A-Za-z0-9]{16,64})/stream/?$ {
    rewrite ^ /d/stream.php?token=$1 last;
}

location ~ ^/d/([A-Za-z0-9]{16,64})/?$ {
    rewrite ^ /d/index.php?token=$1 last;
}

location = /d {
    rewrite ^ /d/guard.php last;
}

location = /d/ {
    rewrite ^ /d/guard.php last;
}

# 站点通用伪静态（你当前的配置）
location / {
    try_files $uri $uri/ $uri.php$is_args$args;
}
```

### 说明

| 路径 | 实际处理 |
|------|----------|
| `/d/` | 安全提醒页 `d/guard.php` |
| `/d/{32位token}` | 分享页 `d/index.php?token=...` |
| `/d/{token}/stream?file=ID` | 中转下载 `d/stream.php` |

分享页链接 **不需要** `?token=` 查询参数，token 在路径中即可。

---

## Apache

项目根目录 `.htaccess` 已包含分享路由；需启用 `mod_rewrite` 且 `AllowOverride All`。

`d/.htaccess` 负责目录内 token 解析。

---

## 验证

1. 访问 `https://你的域名/d/` → 应显示「安全访问提醒」
2. 创建分享后访问 `https://你的域名/d/{token}` → 应打开分享页（非 404）
3. PHP 代码内 `ShareRouter::parseToken()` 会在 rewrite 未生效时尝试从 `REQUEST_URI` 解析（兜底）

---

## 常见问题

**Q：为什么 `try_files`  alone 不够？**  
`/d/abc123...` 在磁盘上不存在对应文件或目录，Nginx 直接返回 404，不会落到 PHP。

**Q：子目录部署可以吗？**  
可以。确保 rewrite 目标路径相对于站点根目录（如 `/subdir/d/index.php`）。

**Q：还需要改 PHP 吗？**  
一般不需要；配置好 Nginx/Apache 后解压覆盖即可。
