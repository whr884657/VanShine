# VanShine 伪静态 / URL 重写配置

VanShine 分享短链接格式为 `{域名}/d/{token}`。若未正确配置，访问分享链接会出现 **404**。

---

## Nginx（推荐）

只需 **两条** `location`：保留站点通用规则，并为 `/d` 增加一条 fallback 到 `d/index.php`。

```nginx
# 分享短链接：统一由 d/index.php 解析 token（含 /stream 下载）
location ^~ /d {
    try_files $uri $uri/ /d/index.php$is_args$args;
}

# 站点通用伪静态
location / {
    try_files $uri $uri/ $uri.php$is_args$args;
}
```

### 说明

| 访问路径 | 实际处理 |
|----------|----------|
| `/d` 或 `/d/` | `d/index.php` → 无 token 时显示安全提醒 |
| `/d/{token}` | `d/index.php` → 分享页 |
| `/d/{token}/stream?file=ID` | `d/index.php` → 文件流下载/预览 |

`index.php` 会从 URL 中自动识别 token，**无需** 多条 rewrite 分别指向 `guard.php`、`stream.php`。

---

## Apache

根目录 `.htaccess` 与 `d/.htaccess` 已配置为统一入口 `d/index.php`；需启用 `mod_rewrite` 且 `AllowOverride All`。

---

## 验证

1. 访问 `https://你的域名/d/` → 应显示「安全访问提醒」（不应报 VS_ROOT 重复定义）
2. 创建分享后访问 `https://你的域名/d/{token}` → 应打开分享页
3. 下载链接 `/d/{token}/stream?file=ID&download=1` 应正常下载

---

## 常见问题

**Q：为什么只有 `try_files` 不够？**  
`/d/abc123...` 在磁盘上不存在对应文件，Nginx 直接 404，需 fallback 到 `d/index.php`。

**Q：子目录部署可以吗？**  
可以。`try_files` 最后一项路径相对于站点根目录即可。
